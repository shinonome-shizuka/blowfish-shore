## 重构目标

### 1. 文件拆分
```
assets/css/
├── custom.css (主入口，仅导入)
├── tokens.css (设计令牌)
├── components/
│   ├── card.css (卡片组件)
│   ├── hero.css (Hero区域)
│   ├── timeline.css (时间轴)
│   ├── podcast.css (播客播放器)
│   └── navigation.css (导航、页脚)
├── effects/
│   ├── steamwave.css (蒸汽波网格)
│   └── dotfield.css (DotField效果)
└── dark-mode.css (暗色模式统一)
```

### 2. 合并同类项
- 所有卡片样式合并到 `components/card.css`
- 所有暗色模式样式合并到 `dark-mode.css`
- 所有响应式样式按断点分组

### 3. 统一命名规范
```css
/* BEM 命名：[block]__[element]--[modifier] */
.card { }
.card__title { }
.card__meta { }
.card--featured { }

/* 暗色模式：统一使用 [data-theme="dark"] */
[data-theme="dark"] .card { }
```

## 风险控制

### 风险点
1. **样式丢失** - 拆分过程中可能遗漏
2. **优先级冲突** - 文件顺序影响 CSS 优先级
3. **暗色模式破坏** - 暗色模式样式不完整
4. **响应式问题** - 媒体查询丢失

### 缓解措施
1. 每次拆分后完整测试所有页面
2. 保持 `custom.css` 作为主入口，控制导入顺序
3. 使用 Git 分支进行重构，便于回滚
4. 建立视觉回归测试
